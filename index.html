Q&A Session

Technical Questions

üîß Technical Implementation

Q1: Why did you choose a smartphone-based system instead of desktop? +

Answer: We chose smartphones for three main reasons:

  • Cost-effectiveness: Most small business owners already have smartphones
  • Portability: Staff can move around the store while accessing the system
  • Familiarity: Everyone knows how to use smartphones, minimizing training needs
Q2: How does the system work without internet connection? +

Answer: We designed it as an offline-first application. All data is stored locally on the device using SQLite database. When the phone is connected to Wi-Fi, optional backup features can sync data, but daily operations require no internet.

Q3: What programming language and tools did you use? +

Answer: We used:

  • Frontend: Java with Android Studio
  • Database: SQLite for local storage
  • Reporting: Built-in PDF generation libraries
  • Version Control: GitHub for team collaboration
Q4: How do you ensure data security? +

Answer: Multiple security layers:

  • User Authentication: Password-protected login
  • Role-Based Access: Different permissions for owner vs cashier
  • Local Storage: Data stays on the device, not on clouds
  • Backup Option: Manual backup to external storage
Q5: What happens if the smartphone gets lost or damaged? +

Answer: We implemented:

  • Regular backup reminders to save data to Google Drive or SD card
  • Multiple user accounts so losing one phone doesn't compromise all data
  • Data export feature to download reports regularly

Q&A Session

Functionality Questions

‚öôÔ∏è System Functionality

Q6: How does real-time inventory tracking work? +

Answer: Every sale triggers an automatic stock deduction. When a cashier processes a sale:

  1. System checks current stock
  2. Processes the sale
  3. Automatically updates stock levels
  4. If stock reaches minimum level, shows red alert
Q7: Can the system handle multiple cashiers? +

Answer: Our current version supports single-store, multi-user but on one device at a time. For multiple cashiers simultaneously:

  • Designated sales smartphone at counter
  • All cashiers use same device with their login credentials
  • System tracks who processed each sale
Q8: How do you handle returns or refunds? +

Answer: Simple 3-step process:

  1. Find original transaction in "Sales History"
  2. Select items to return
  3. System automatically:
    • Adjusts inventory (adds back returned items)
    • Processes refund amount
    • Generates return receipt
    • Updates financial records
Q9: What types of reports can the owner generate? +

Answer: Four main report categories:

  • Sales Reports: Daily/Weekly/Monthly totals, top-selling items
  • Inventory Reports: Current stock, low stock alerts, stock value
  • Financial Reports: Profit/Loss, expenses, cash flow
  • Customer Reports: Purchase history, frequent buyers
Q10: How does the system calculate profit? +

Answer: Automatic profit calculation using:

Profit = Selling Price - Cost Price

Where:

  • Selling Price: Price customer pays
  • Cost Price: Price store paid supplier (entered during product setup)
  • System tracks both for every item sold
